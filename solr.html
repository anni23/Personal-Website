Data Indexing - Indexing large volume of data in a Solr instance, in this case tweets collected from Twitter.

Solr – Solr is an open source search platform written in java.

There are two important phases in this project
1) Indexing
2) Querying

Indexing – In this phase, you feed in all the information. Basic unit of information in solr is a document. In solr world, documents are composed of fields. When you index a document into solr it takes the information in the document’s fields and creates a term document index. The managed-schema.xml is the the place where you tell solr which fields should be indexed and how. You can create an index with one single field or you can even build an index with many different fields. In managed-schema.xml, you can specify different processing like tokenizing, stemming, removing stop words, etc. to be done while indexing.

Querying – In this phase, you ask questions. When you perform query, solr can quickly consult the index and return the matching documents. In managed-schema.xml, you can specify different operations like tokenizing, stemming, removing stop words, querying synonyms, etc. to be performed at query time.

Implementation Steps:
1) Configured an AWS account and a simple EC2 instance.
2) Used the Twitter’s Rest API and a Java based Twitter client to query Twitter and collect tweets.
3) Used the org.json library to for preprocessing the retrieved tweets. Configured a Solr instance to correctly index the collected tweets as per the project requirements.
4) Hosted a web application and connected it to Solr instance.
